MySQL_slave不能同步更新

By Mars (hnynes)

发表于 2008-04-26

mysql slave server 上进程停止, 重启后不接收master上的更新

    mysql>show slave status\G;
    *************************** 1. row ***************************
             Slave_IO_State: Waiting for master to send event
                Master_Host: 192.168.10.168
                Master_User: admin
                Master_Port: 3306
              Connect_Retry: 60
            Master_Log_File: mysql-bin.000055
        Read_Master_Log_Pos: 720589499
             Relay_Log_File: YD227-relay-bin.000108
              Relay_Log_Pos: 505783744
      Relay_Master_Log_File: mysql-bin.000051
           Slave_IO_Running: Yes
          Slave_SQL_Running: No
            Replicate_Do_DB: try2u
        Replicate_Ignore_DB:
         Replicate_Do_Table:
     Replicate_Ignore_Table:
    Replicate_Wild_Do_Table:
    Replicate_Wild_Ignore_Table:
                 Last_Errno: 1062
                 Last_Error: Error 'Duplicate entry '370' for key 4' on query. Default database: 'webdb'. Query: 'INSERT INTO webdb.t_coin_update SET spid = 370 ,   uid = 1863421 , got = 0 , room_id = 235789'

错误找到： Slave_SQL_Running: No
google到原因:
1.程序可能在slave上进行了写操作
2.也可能是slave机器重起后，事务回滚造成的.

明显是第二个原因造成, 解决办法:

mysql5.6以前的版本
slave stop;
set GLOBAL SQL_SLAVE_SKIP_COUNTER=1;
slave start;

mysql5.6以后的版本
只能人工确定数据, 并手动干预.
如果数据量比较小的话, 建议重新做slave实例.

几个跟热备有关的mysql命令:(需要在mysql命令行界面执行）

stop slave #停止同步
start slave #开始同步，从日志终止的位置开始更新。
SET SQL_LOG_BIN=0|1 #主机端运行，需要super权限，用来开停日志，随意开停，可能会造成主机从机数据不一致. 一般为seddsion级别临时操作, 不需要进行复制到slave实例的操作
SET GLOBAL SQL_SLAVE_SKIP_COUNTER=n # 客户端运行，用来跳过几个事件，只有当同步进程出现错误而停止的时候才可以执行。 mysql5.6以前的版本可如此使用
RESET MASTER #主机端运行,清除所有的日志，这条命令就是原来的FLUSH MASTER
RESET SLAVE [all] #从机运行，清除日志同步位置标志，并重新生成master.info

LOAD TABLE tblname FROM MASTER #从机运行，从主机端重读指定的表的数据，每次只能读取一个，受timeout时间限制，需要调整timeout时间。执行这个命令需要同步账号有 reload和super权限。以及对相应的库有select权限。如果表比较大，要增加net_read_timeout 和 net_write_timeout的值
LOAD DATA FROM MASTER #备机执行，从主机端重新读入所有的数据。执行这个命令需要同步账号有reload和super权限。以及对相应的库有select权限。如果表比较大，要增加net_read_timeout 和 net_write_timeout的值
CHANGE MASTER TO master_def_list #在线改变一些主机设置，多个用逗号间隔,比如
CHANGE MASTER TO
[MASTER_HOST=’host_name’,] #主机名
[MASTER_USER=’user_name’,] #连接数据库的用户名
[MASTER_PASSWORD=’password’,] #连接数据库的用户名对应的密码
[MASTER_PORT = port_num,] #主机数据库端口
[MASTER_LOG_FILE = ‘master_log_name’,] #指定当前准备开始同步的主机的日志文件名
[MASTER_LOG_POS = master_log_pos,]; #指定文件中位置
MASTER_POS_WAIT() #备机执行
SHOW MASTER STATUS #主机执行，查看日志位置信息
SHOW SLAVE HOSTS #主机执行，看连入的备机的信息。
SHOW SLAVE STATUS # 查看同步状态信息
SHOW MASTER LOGS # 查看当前存留的binary log的信息
SHOW BINLOG EVENTS [ IN ‘logname’ ] [ FROM pos ] [ LIMIT [offset,] rows ] # 查看binlog event日志信息
PURGE [MASTER] LOGS TO ‘logname’ ; PURGE [MASTER] LOGS BEFORE ‘date’ # 清理binlog日志至logname或是某个时间
